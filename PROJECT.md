# プロジェクト概要：家具配置シミュレーター

## 目的
引越しや家具購入の前に、部屋に家具が収まるか・予算内に収まるかを
シミュレーションできるWebアプリ。ポートフォリオ用途。

## ターゲットユーザー
引越しが決まり、これから家具を買うカップル。
「予算」と「家具のサイズ」を同時に考えながら配置を検討する。

## コア体験（この一本線のみ。これ以外は作らない）
1. 部屋のサイズ（縦・横 cm）を入力する
2. 2Dキャンバス（真上視点）に部屋が表示される
3. まず実物を参照せず、プリセットの「家具の枠」（種類＋サイズ）を配置する
4. 全体予算を設定する
5. 予算内に収まる実商品の組み合わせを提案する（楽天市場API＋予算配分アルゴリズム。有料LLMは使わない）
6. 提案に対して要望（安め優先／スタイル等）を加えて再提案する

## 旧コンセプト（廃止）
「実商品を検索して置く → 合計金額表示」の流れは廃止し、上記の
「間取り → 予算 → 提案 → 要望」の流れに作り替えた。

## 発展フェーズ（コア体験の後に拡張。TASKS.md Phase 6以降）
- 手持ち家具の考慮とデータ出力
- 生活動線の自動計算と可視化
- 2D / 3D のシームレスなプレビュー切り替え
- 非矩形（L字型など）ルーム対応と間取り図スキャン

## 作らないもの（スコープ外）
- 共同編集・アカウント機能

## 開発方針
- 自律ループ（/loop）が TASKS.md のバックログを上から順に1タスクずつ自動着手してよい
- ただし以下に該当する場合は必ず一度停止し、人間の確認を取ってから再開すること（先回りして進み続けない）:
  - TASKS.md内のフェーズ境界に到達したとき
  - このファイルの「作らないもの」に触れる可能性があるとき
  - TASKS.mdに書かれていない設計・UI/UX判断が必要なとき
- 開発ルールはCLAUDE.mdに従うこと（自律ループの詳細手順はCLAUDE.md「自律ループ運用」を参照）
