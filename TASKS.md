## Phase 5: 入口・窓の考慮

設計メモ（この前提でタスクを進める。前提を変える判断が必要なときは停止して確認）:
- 入口・窓は部屋の「壁の辺」に置くマーカー。データは { wall: "top"|"bottom"|"left"|"right", offsetCm(辺の始点からの中心位置), widthCm, kind: "door"|"window" }。真上視点なので壁の辺上に線分として描画する。
- 入口(door)は室内側に通行/開閉クリアランス（既定 60cm）を確保し、その帯には家具を置けない。
- 窓(window)はクリアランス不要。塞いでいる場合は警告のみ（配置は禁止しない）。
- 入口・窓・クリアランス設定も localStorage に保存する。

- [x] T-開口-1: 壁（上下左右）に入口/窓のマーカーを追加・削除するUIと、キャンバス上への描画（辺・中心位置・幅・種別を指定）
- [x] T-開口-2: 入口前のクリアランス帯（既定60cm）を定義し、家具の当たり判定と自動配置(findFreePosition)がその帯を避けるように反映
- [x] T-開口-3: 窓の前に家具が重なっている場合、一覧または提案で警告表示（配置は許可のまま）
