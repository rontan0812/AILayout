## Phase 5: 入口・窓の考慮

設計メモ（この前提でタスクを進める。前提を変える判断が必要なときは停止して確認）:
- 入口・窓は部屋の「壁の辺」に置くマーカー。データは { wall: "top"|"bottom"|"left"|"right", offsetCm(辺の始点からの中心位置), widthCm, kind: "door"|"window" }。真上視点なので壁の辺上に線分として描画する。
- 入口(door)は室内側に通行/開閉クリアランス（既定 60cm）を確保し、その帯には家具を置けない。
- 窓(window)はクリアランス不要。塞いでいる場合は警告のみ（配置は禁止しない）。
- 入口・窓・クリアランス設定も localStorage に保存する。

- [x] T-開口-1: 壁（上下左右）に入口/窓のマーカーを追加・削除するUIと、キャンバス上への描画（辺・中心位置・幅・種別を指定）
- [x] T-開口-2: 入口前のクリアランス帯（既定60cm）を定義し、家具の当たり判定と自動配置(findFreePosition)がその帯を避けるように反映
- [x] T-開口-3: 窓の前に家具が重なっている場合、一覧または提案で警告表示（配置は許可のまま）

## Phase 6: 手持ち家具の考慮とデータ出力

設計メモ:
- 手持ち家具は「所有フラグ付きの家具枠」。提案（購入候補）の対象外、予算計算にも含めない。プリセット/自由サイズで登録し、キャンバス上は所有と分かる見た目にする。
- データ出力は、間取り・配置・開口部・提案を JSON でエクスポート/インポート、提案は買い物リスト（商品名・価格・リンク・合計）としても出せる。

- [x] T-手持-1: 手持ち家具を登録（種類・サイズ・所有フラグ）し、キャンバスで所有と分かる表示にする
- [x] T-手持-2: 所有家具を提案対象外・予算計算対象外にする
- [x] T-出力-1: 間取り・配置・開口部・提案を JSON でエクスポート／インポートする
- [x] T-出力-2: 提案結果を買い物リスト（商品名・価格・リンク・合計）としてテキスト/CSVで出力する

## Phase 10: 設置したいもの・予算からの自動レイアウト提案

設計メモ:
- 真上視点2Dで、家具枠を部屋形状・開口部（入口クリアランス／窓前）・欠け領域を避け、重ならないよう自動配置する。壁付け優先のヒューリスティック。
- 入口は「家具を選んで自動配置」と「予算だけでおまかせ」の2つを用意する（別々の機能）。
- 予算は品目・数量の選定にも使う（予算内に収まる定番構成を組んでから配置）。
- 生成レイアウトは placedItems を置き換え、既存の手動編集・商品提案とデータを共有する。所有家具（owned）は残して避ける。

- [x] T-自動-1: 自動レイアウトエンジン（家具リスト＋部屋形状＋開口部→重なり無し・クリアランス回避の壁付け配置）を純関数で実装し、数値検証する
- [ ] T-自動-2: 「家具を選んで自動配置」UI（家具タイプ＋数量を選び、自動配置を実行してキャンバスへ反映）
- [ ] T-自動-3: 「予算だけでおまかせ」UI（予算と部屋サイズから定番構成の品目・数量を自動選定して配置）
