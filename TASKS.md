## Phase 5: 入口・窓の考慮

設計メモ（この前提でタスクを進める。前提を変える判断が必要なときは停止して確認）:
- 入口・窓は部屋の「壁の辺」に置くマーカー。データは { wall: "top"|"bottom"|"left"|"right", offsetCm(辺の始点からの中心位置), widthCm, kind: "door"|"window" }。真上視点なので壁の辺上に線分として描画する。
- 入口(door)は室内側に通行/開閉クリアランス（既定 60cm）を確保し、その帯には家具を置けない。
- 窓(window)はクリアランス不要。塞いでいる場合は警告のみ（配置は禁止しない）。
- 入口・窓・クリアランス設定も localStorage に保存する。

- [x] T-開口-1: 壁（上下左右）に入口/窓のマーカーを追加・削除するUIと、キャンバス上への描画（辺・中心位置・幅・種別を指定）
- [x] T-開口-2: 入口前のクリアランス帯（既定60cm）を定義し、家具の当たり判定と自動配置(findFreePosition)がその帯を避けるように反映
- [x] T-開口-3: 窓の前に家具が重なっている場合、一覧または提案で警告表示（配置は許可のまま）

## Phase 10: 設置したいもの・予算からの自動レイアウト提案

設計メモ:
- 真上視点2Dで、家具枠を部屋形状・開口部（入口クリアランス／窓前）・欠け領域を避け、重ならないよう自動配置する。壁付け優先のヒューリスティック。

## Phase 12: 方角・採光・影・部屋の照明

設計メモ:
- 真上視点2D・クライアント完結の簡易ライティングモデル。部屋に「北向き（方角）」と「時間帯」を設定する。
- 自然光: 各窓の向き（方角）×時間帯（朝→東, 昼→南, 夕→西）で明るさと入射方向を重み付け。家具が光を遮る範囲を影とする。
- 人工照明: 天井灯（遮蔽なし・広範囲）／フロアランプ（家具で遮蔽）を設置し、自然光に合成する。
- 採光マップはグリッドで計算し、キャンバスに明暗ヒートマップとして可視化（トグル）。採点にも反映（暗がりの減点、デスクの採光など）。

- [x] T-光-1: 方角（北向き）と時間帯の設定UI＋コンパス表示、localStorage保存
- [x] T-光-2: 採光マップ計算（窓の自然光を方角×時間で重み付け＋家具の影）を純関数で実装し数値検証する
- [x] T-光-3: 採光マップをキャンバスに可視化（明暗ヒートマップ、トグル）
- [x] T-光-4: 部屋の照明（天井灯/フロアランプ）の設置UIと、人工光の採光マップへの合成
- [x] T-光-5: 採光・暗がりを採点に反映（暗い領域の減点、デスクの採光など）

## Phase 15: 共有ルームによる同時編集

設計メモ:
- 共有URL（?room=ID）で同じ部屋にアクセスした人が、ほぼ同時に同じ間取り・配置を編集できる。
- 同期方式はサーバー保存＋短間隔ポーリング。状態はサーバーのインメモリに保持（バージョン管理でLast-Write-Wins）。※Vercelサーバーレスでは単一インスタンス前提の制約あり。
- 共有ドキュメントは localStorage と同じ内容（roomSize/roomShape/placedItems/openings/budget/northDeg/timeOfDay/lights）。
- アカウント不要。クライアントIDでプレゼンス（参加人数）を集計。

- [x] T-共有-1: ルーム状態のAPI（GET取得/PUT更新・バージョン・プレゼンス集計）をインメモリで実装する
- [x] T-共有-2: 共有URLでのルーム作成/参加とポーリング同期（ローカル⇄リモート、エコー防止）
- [ ] T-共有-3: 参加人数の表示と共有バナー（URLコピー・退出）
